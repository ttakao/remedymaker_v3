# システム仕様書：レメディ生成機コントロールシステム (Version 3.1)

## 1. システム概要
本システムは、USB接続されたレメディ生成機（Arduinoベース）をブラウザから直接制御するためのWebアプリケーションである。
Google Chromeブラウザが動作するあらゆる環境（Windows、macOS、ChromeOSなど）で、特別なドライバー不要で利用可能とする。

### 1-1. 基本方針
*   **脱フレームワーク**: 外部ライブラリに依存せず、素のJavaScript (Vanilla JS) で構築し、長期的な保守性を確保する。
*   **Web Serial APIの採用**: ブラウザから直接USBシリアルポート（Arduino）を制御する。
*   **サーバー構成**: 一般的なレンタルサーバー（PHP 7.4 / 8.x + MySQL）で動作し、SPA（Single Page Application）形式で動作。APIとのやり取りはJSON形式で行う。
*   **ハードウェア互換性**: 既存のArduinoファームウェア（BaudRate 115200、セミコロン終端、whoコマンドによる応答）に適合した送信シーケンスを実装する。

## 3. 認証・セキュリティ処理 (A)

### A-1. ログインフロー
1.  ユーザーはメールアドレスを入力し「パスキー送信」ボタンを押す。
2.  サーバー側（PHP）はアカウントがロックされていないことを確認の上、8桁の数字ランダム文字列を生成。
3.  登録されているメールアドレス宛にPHPの `mb_send_mail()` を用いてパスキーを送信。同時に `password_hash()` でハッシュ化して `users.passkey_hash` に保存し、`update_date` を更新。ログイン失敗回数（`login_attempts`）を0にリセットする。
4.  画面にパスキー入力フォームを表示。

### A-2. 認証判定とロックアウト
1.  入力があった際、サーバーは `locked_until` をチェックし、現在時刻より後の場合は「ロック中」のエラーを返す。
2.  `update_date` から5分以内であることを確認。
3.  入力されたパスキーを `password_verify()` で照合。
4.  **照合成功時**:
    *   `login_attempts` を `0` に、`locked_until` を `NULL` にクリア。
    *   セッションを開始し、認証済みCookieを発行する（Cookieの寿命は12時間目標。ただし、サーバーのガベージコレクション事情等により早期切断される可能性がある旨の注意書きをログイン画面等に明記する）。
5.  **照合失敗時**:
    *   `login_attempts` を `+1` する。
    *   失敗回数が `3` 回に達した場合、`locked_until` に「現在時刻 + 30分」をセットする。

### A-3. ログアウト処理
*   画面上の「ログアウト」ボタンを押すことで、サーバー側のセッションを破棄し、ブラウザのセッションCookieを明示的に削除して初期画面（ログイン画面）へ遷移する。

## 4. データ検索・表示処理 (B/C)

### B-1. 初期ロード
*   ログイン成功後、`index.html` はサーバーから以下のデータをJSONで取得し、ブラウザのメモリ上に一括保持する（約300件程度のデータ規模を想定）。
    *   ユーザーが閲覧権限を持つグループ・サブグループ・周波数リスト。
    *   ユーザー自身のカスタムリスト。

### B-2. 画面構成 (SPA形式)
*   **メインエリア**:
    *   グループリスト（スクローラブル・テーブル）
    *   サブグループリスト（スクローラブル・テーブル）
    *   周波数リスト（スクローラブル・テーブル）
    *   文字列検索窓（jname, enameをリアルタイムフィルタリング）
*   **実行エリア**:
    *   「処理する周波数リスト」表示エリア（テキスト）
    *   「接続設定」ボタン（設定ダイアログを表示。初回のみ必須）
    *   「書き込み（開始）」ボタン
    *   「中止」ボタン
    *   **進捗表示インジケータ**: 現在送信中の周波数、全体のプログレスバー、該当周波数の残り待機秒数のカウントダウンを表示する。

---

## 5. データ転送処理 (D)

### D-1. 接続設定ダイアログ
「接続設定」ボタンを押すと、以下の項目を持つダイアログを表示する。
1.  **シリアルポート選択**: `navigator.serial.requestPort()` を呼び出すための「デバイスを選択」ボタン。
2.  **待機秒数の指定**: ひとつの周波数を継続する時間（秒）を入力するテキストフィールド（デフォルト値: 20秒）。
3.  **設定保存**: 選択されたポート情報と入力された秒数を変数に保持し、ダイアログを閉じる。
    *(※利用者の誤解を防ぐため、ページの再読み込み時にポートを自動的に再接続する機能は設けない。接続は毎回明示的に行わせる)*

### D-2. 周波数送出シーケンス（Arduino互換仕様）
「書き込み（開始）」ボタンが押された際の処理：
1.  ポートが未接続、または待機秒数が不正な場合はエラーを表示する。
2.  **ハンドシェイク（疎通確認）**:
    *   シリアルポートを開く（**BaudRate: 115200**）。
    *   デバイスへ `"who;"` （セミコロン終端）を送信。
    *   受信バッファを監視し、デバイスから `"AD9833\r\n"` が返ってくるか確認。
    *   一致しない、または一定時間内に返答がない場合は「レメディ生成機が正しく接続されていません」と表示して処理を中断する。
3.  処理する周波数リスト（カンマ区切り文字列）を配列に変換。
4.  ループ処理を開始：
    *   配列から1つの周波数文字列（例: `"12.34"`) を取り出す。
    *   シリアルポートへ末尾にセミコロンを付加した文字列（例: `"12.34;"`）として送信。
    *   デバイスから `"12.34\r\n"` のエコーバックを受信したことを確認して送信完了と判定。
    *   **待機**: 設定された秒数分、非同期（`Promise` と `setTimeout` によるカウントダウン）で待機。この間、画面に進捗状況をリアルタイムで表示する。
    *   次の周波数へ。
5.  全リスト終了後、ポートをクローズし、画面に「終了」ダイアログを表示する。

### D-3. 中止処理
*   送信ループの実行中に「中止」ボタンが押された場合：
    1.  ループのフラグを検知し、即座に送信ループをブレイク。
    2.  デバイスへ `"0;"`（出力停止コマンド）を送信。
    3.  ポートをクローズし、待機表示をクリアする。

## 6. カスタムデータ処理 (E)
*   ユーザーは自分専用の周波数リストを登録・編集・削除できる。
*   保存先は `user_custom_frequencies` テーブル。
*   データ入力時、フロントエンド（JavaScript）側で「半角数字、小数点、および区切り用のカンマ」以外の文字列が含まれていないか厳密にバリデーション（正規表現によるチェック）を行う。

---

## 7. 管理機能 (F)
管理者権限（`role = 'admin'`）を持つユーザーのみがアクセス可能な管理画面（別メニューまたは管理モード）。デザインはシンプルなシステム標準コントロールで構成する。

### F-1. ユーザーメンテナンス
*   ユーザーの新規作成（名前、メールアドレス、権限レベルを設定）。
*   ユーザー情報の編集・削除、および支払いフラグのON/OFF。
*   ロックアウト（3回失敗）されたアカウントの強制解除（失敗回数をリセット、ロック日時をクリアするボタン）。

### F-2. 周波数マスタメンテナンス
グループ、サブグループ、周波数リストの整合性を保ちながらCRUD操作を行うためのダイアログ画面。
*   **グループ登録/編集**: グループ名、対象研修IDの編集。
*   **サブグループ登録/編集**: 親グループをプルダウン選択し、サブグループ名を追加・編集。
*   **周波数登録/編集**: 親グループ・サブグループを絞り込み選択したうえで、日本語名称、英語名称、カンマ区切り周波数リスト（半角数字・小数点・カンマ）を入力して登録・編集。

## 8. 技術仕様詳細

### 8-1. フロントエンド
*   **HTML/CSS/JS**: `index.html` 1ファイル、または最小限のファイル構成。
*   **JavaScript**: Vanilla JS (ES6+)。API通信には `fetch`、シリアル通信には Web Serial API の `ReadableStream/WritableStream` 処理を組み込む。

### 8-2. バックエンド
*   **言語**: PHP 7.4 / 8.x
*   **セキュリティ**: SQLインジェクション対策（プリペアドステートメント）、CSRF対策トークンの埋め込み。
*   **APIエンドポイント**: `api.php` を設置し、`action` パラメータで処理を分岐。

---

## 9. 特記事項
*   本仕様はWeb標準技術で完結しているため、将来的なOSのアップデート等に強く影響を受けない構造となっている。
*   「中止」時に周波数 `0` を送出する機能を追加したことで、意図しない波形出力の持続をシステム側で最小限に抑えるよう配慮している。
